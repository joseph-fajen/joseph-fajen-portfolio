## AI-Assisted Documentation Workflows

<AccordionGroup>
  <Accordion title="Adversarial quality gates">
    For the Book of Archon, I designed a Writer Agent vs. Attacker Agent pipeline.
    The Attacker challenged every factual claim; the Writer had to defend or revise.
    Quality gates set at 7/10+ before content could advance. The documentation was
    accepted as the official user guide for a 20k-star project.
  </Accordion>
  <Accordion title="Hallucination testing protocol">
    For the Marlowe AI assistant, I developed an adversarial testing framework specifically
    designed to surface hallucinations. Ran 20+ prompt engineering iterations against
    test queries designed to break the model's accuracy. No hallucinations detected
    during testing.
  </Accordion>
  <Accordion title="Voice learning system">
    My personal agent system learns from approved content. Writing samples and session
    transcripts feed into a voice profile, which is used to humanize AI-drafted content.
    Gets more accurate over time.
  </Accordion>
</AccordionGroup>

## Specification-Driven AI Execution

For the Book of Archon, I developed a full methodology for AI-assisted documentation production. The approach:

1. **Human writes the PRD** — detailed spec encoding structure, voice, terminology, acceptance criteria
2. **Workflow executes against spec** — 31-node DAG with per-chapter validation gates
3. **AI drafts each section** — consistent execution against the blueprint
4. **Human edits and approves** — judgment, polish, nuance

**Output:** 10 chapters, 11,000 words, 6 hours total — versus an estimated 32-40 hours traditionally. The output was 80-90% production-ready before editing.

## Tooling philosophy
