## How I work

Target audience analysis matters more than many people realize. I spend time understanding who the docs are for before I write them—different user types need different entry points and different levels of detail.

My favorite context is being embedded in an engineering team. I cultivate relationships with developers so I can accurately reflect the nuances of the project in the documentation. Weekly syncs, iterative PR reviews, catching issues before release.
